SET-A Q & A

  • 1. How to create reactive forms?

    Answer: To create reactive forms, first, import ReactiveFormsModule in your Angular module. Then, use FormBuilder and FormGroup to define and manage form controls. Reactive forms are useful for handling complex form logic and validations.

  • 2. How to add built-in validation for email in reactive form?

    Answer: Angular provides a built-in validator Validators.email that can be used for validating email input in a reactive form.

  • 3. How to add built-in validation for required fields in reactive form?

    Answer: The Validators.required built-in validator ensures that the input field must not be empty.

  • 4. How to add regex validation for reactive forms?

    Answer: To add regex validation, you can use Validators.pattern and pass a regular expression.

  • 5. How to generate routing?

    Answer: To generate routing, run ng generate module app-routing --flat --module=app. This creates a routing module that you can configure in app-routing.module.ts.

  • 6. How to configure routes with parameters?

    Answer: To configure routes with parameters, use the :parameter syntax in the path. You can access the parameters using ActivatedRoute.

  • 7. What is DebounceTime in RxJS?

    Answer: debounceTime is an RxJS operator that waits for a specified period after the last emitted value before emitting the final value. It is useful for preventing multiple requests in quick succession.

  • 8. What is the pipe operator?

    Answer: The pipe operator in RxJS allows you to chain multiple operators together. It's used to transform, filter, or manage the data flow.

  • 9. What is the distinctUntilChanged operator?

    Answer: The distinctUntilChanged operator ensures that only distinct values (values that are different from the previous value) are emitted, which is useful in scenarios where you want to avoid redundant processing.

  • 10. What is the filter operator in RxJS?

    Answer: The filter operator allows you to emit values from an observable based on a condition, filtering out unwanted values.
